125 ### 2.8.0
126  * __197573:__ Suppressed extraneous \ref RAIL_EVENT_TX_START_CCA events that
127  might occur during long CCA durations. Now only one such event should occur
128  per CCA try.
129  * __295265:__ Added two new APIs, \ref RAIL_GetSyncWords and \ref
130  RAIL_ConfigSyncWords(), to allow getting and setting the sync word
131  configuration of your PHY at runtime.
132  * __301472:__ On EFR32xG12 thru EFR32xG14, added support for 802.15.4G-2012
133  SUN PHY dynamic frame payload whitening on reception and transmit based on
134  the PHY header's Data Whitening flag setting. This feature is automatically
135  enabled when \ref RAIL_IEEE802154_ConfigGOptions()' \ref
136  RAIL_IEEE802154_G_OPTION_GB868 is enabled, and assumes the radio
137  configuration specifies the appropriate whitening algorithm and settings.
138  * __301488:__ On EFR32xG12 thru EFR32xG14, added support for 802.15.4G-2012
139  SUN PHY dynamic frame payload 2/4-byte FCS (CRC) on reception and transmit
140  based on the PHY header's FCS Type flag setting. This feature is
141  automatically enabled when \ref RAIL_IEEE802154_ConfigGOptions()' \ref
142  RAIL_IEEE802154_G_OPTION_GB868 is enabled. The radio configuration's
143  (single) CRC algorithm settings are ignored, overridden by RAIL.
144  * __301501:__ On EFR32xG12 thru EFR32xG14, 802.15.4 AutoACK behavior has also
145  been updated so transmitted immediate ACKs reflect the Whitening and
146  2/4-byte FCS of the received frame being acknowledged.
147  * __369722:__ Added \ref RAIL_TX_OPTION_CCA_ONLY to just perform CCA
148  (CSMA/LBT), stopping short of automatically transmitting when the channel is
149  clear.
150  * __369727:__ Added support for a new \ref RAIL_EVENT_TX_STARTED, triggered
151  when the first preamble bit is about to go on-air. Also included the ability
152  to retrieve the equivalent \ref RAIL_PACKET_TIME_AT_PREAMBLE_START timestamp
153  of that event from the event's handler via \ref
154  RAIL_GetTxTimePreambleStart(). Note: This new event shifted the bit
155  positions of some events in \ref RAIL_Events_t.
156  * __369727:__ Also changed \ref RAIL_GetRxTimePreambleStartAlt(), \ref
157  RAIL_GetRxTimeSyncWordEndAlt(), and \ref RAIL_GetRxTimeFrameEndAlt() to
158  properly update its pPacketDetails' \ref
159  RAIL_PacketTimeStamp_t::timePosition to reflect the adjusted \ref
160  RAIL_PacketTimeStamp_t::packetTime rather than leaving it as \ref
161  RAIL_PACKET_TIME_DEFAULT.
162  * __384878:__ Clarified documentation of the \ref RAIL_EVENT_RX_ACK_TIMEOUT
163  event and \ref RAIL_AutoAckConfig_t::ackTimeout period which extends only to
164  packet sync word detection of an expected ACK, not packet completion of that
165  ACK.
166  * __397072:__ Added an API, \ref RAIL_StopInfinitePreambleTx, that can stop an
167  infinite preamble on PHYs configured to use infinite preambles.
168  * __402991:__ Added additional information to the packet trace stream for the
169  Z-Wave protocol to indicate what region is currently active to help with
170  decoding.
171  * __406080:__ Added support for RFSENSE Selective (OOK) mode for supported
172  chips, which currently includes only EFR32xG22 devices. Please refer to RAIL
173  internal chip specific documentation for more details.
174  * __406871:__ Documented RAIL's internal 16-packet metadata FIFO which exists
175  on EFR32 platforms supplementing the receive FIFO of packet data. Refer to
176  \ref Data_Management and \ref efr32_main for details. Included is support
177  for a new \ref RAIL_EVENT_RX_FIFO_FULL, triggered with any packet completion

179  the application it must free up the oldest packets/data ASAP to reduce the
180  chance of \ref RAIL_EVENT_RX_FIFO_OVERFLOW (however, overflow may already

182  events in \ref RAIL_Events_t.

184  RAIL_STATUS_INVALID_STATE if called when the radio is not idle, enforcing
185  its documented behavior.
186  * __414114:__ Added a new PA mode which will attempt to automatically choose
187  the PA which consumes the least amount of current to reliably produce the
188  requested output power. See \ref RAIL_EnablePaAutoMode() for details.
189  * __417340:__ Fixed an issue where \ref RAIL_RxPacketDetails_t::isAck would
190  incorrectly be set true for non-ACK or unexpected ACK packets received
191  successfully (e.g. when \ref RAIL_IEEE802154_ACCEPT_ACK_FRAMES is enabled)
192  or aborted while waiting for the expected ACK. Note that when \ref
193  RAIL_RX_OPTION_IGNORE_CRC_ERRORS is in effect, an expected ACK includes one

195  * __418493:__ RAIL_ConfigRadio will now return RAIL_STATUS_INVALID_STATE if

197  returning success but not applying the change.
198  * __427934:__ Fixed a race condition that could cause a device to not re-

201  * __430026:__ Enforced and clarified that RAIL_Init() should not be called
202  more than once per protocol.
203  * __430081:__ Fixed an issue where the first Clear Channel Assessment (CCA)

291  * __401826:__ Fixed several issues and race conditions with
292  \ref RAIL_ScheduleRx() window end handling and event reporting
293  to conform to its intended design:
294  * When the RX window ends with no \ref RAIL_EVENTS_RX_COMPLETION
295  imminent, \ref RAIL_EVENT_RX_SCHEDULED_RX_END is posted.
296  * When the RX window is implicitly ended by one of the
297  \ref RAIL_EVENTS_RX_COMPLETION, e.g. because it results in
298  an RX transition to Idle or because \ref
299  RAIL_ScheduleRxConfig_t::rxTransitionEndSchedule is non-zero,
300  the event(s) posted depend on that setting:
301  * When zero, both events are posted simultaneously.
302  * When non-zero, only the appropriate \ref RAIL_EVENTS_RX_COMPLETION
303  is posted, unless that event is not enabled, in which case
304  \ref RAIL_EVENT_RX_SCHEDULED_RX_END is substituted instead.
305  * When the RX window ends while receiving a packet, it is
306  deferred to the expected RX completion event (which includes
307  aborting that packet when \ref RAIL_ScheduleRxConfig_t::hardWindowEnd
308  is non-zero). Event(s) reported at that time are the same as in the
309  previous case.

406 ### 2.6.3
407  * Fixed an issue introduced in 2.6.2 where issuing a \ref RAIL_Idle() with
408  mode \ref RAIL_IDLE_FORCE_SHUTDOWN_CLEAR_FLAGS could sometimes cause the
