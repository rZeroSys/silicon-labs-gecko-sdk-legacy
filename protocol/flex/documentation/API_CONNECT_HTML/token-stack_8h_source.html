44 // The basic TOKEN_DEF macro should not be used directly since the simplified
45 // definitions are safer to use. For completeness of information, the basic
46 // macro has the following format:
47 //
48 // TOKEN_DEF(name,creator,iscnt,isidx,type,arraysize,...)
49 // name - The root name used for the token
50 // creator - a "creator code" used to uniquely identify the token
51 // iscnt - a boolean flag that is set to identify a counter token
52 // isidx - a boolean flag that is set to identify an indexed token
53 // type - the basic type or typdef of the token
54 // arraysize - the number of elements making up an indexed token
55 // ... - initializers used when reseting the tokens to default values
56 //
57 //
58 // The following convenience macros are used to simplify the definition
59 // process for commonly specified parameters to the basic TOKEN_DEF macro
60 // DEFINE_BASIC_TOKEN(name, type, ...)
61 // DEFINE_INDEXED_TOKEN(name, type, arraysize, ...)
62 // DEFINE_COUNTER_TOKEN(name, type, ...)
63 // DEFINE_FIXED_BASIC_TOKEN(name, type, address, ...)
64 // DEFINE_FIXED_INDEXED_TOKEN(name, type, arraysize, address, ...)
65 // DEFINE_FIXED_COUNTER_TOKEN(name, type, address, ...)
66 // DEFINE_MFG_TOKEN(name, type, address, ...)
67 //
